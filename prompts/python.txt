Обращай внимание на:
- неверные ветвления и условия (if/while), логические ошибки
- ошибки индексов и границ коллекций
- ошибки при работе с None/пустыми значениями
- ошибки обработки исключений и ресурсов (файлы/сокеты)
- уязвимости (инъекции, небезопасные вызовы)

Пример 1:
```python
def find_max(lst):
    max_val = 0
    for x in lst:
        if x > max_val:
            max_val = x
    return max_val
```
Проблема: Неверный результат для списков с отрицательными числами (max_val начинается с 0).
Решение: Инициализировать max_val первым элементом или использовать float('-inf').

Пример 2:
```python
with open(path) as f:
    data = f.read()
print(data[0])
```
Проблема: Возможен IndexError при пустом файле.
Решение: Проверить длину data перед доступом.

Пример 3:
```python
def get_user(users, i):
    return users[i]
```
Проблема: Возможен выход за пределы списка.
Решение: Проверять i на диапазон.

Пример 4:
```python
query = "SELECT * FROM users WHERE name = '%s'" % name
cursor.execute(query)
```
Проблема: SQL-инъекция.
Решение: Использовать параметризованные запросы.
